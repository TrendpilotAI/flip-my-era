## 1. Security

### 1.1 🔴 VITE_OPENAI_API_KEY exposed in client bundle
**File:** `src/modules/shared/utils/env.ts:37` (approx)
```ts
return getEnvVar('VITE_OPENAI_API_KEY');
```
Any `VITE_*` env var is baked into the client-side JS bundle at build time. Every browser visitor can extract this key from DevTools → Network or by reading the bundle.

**Fix:** Remove from Vite env. Create `openai-proxy` edge function. Route all OpenAI calls through it.

### 1.2 🔴 VITE_GROQ_API_KEY in client bundle
**File:** `src/modules/shared/utils/env.ts`, `src/vite-env.d.ts`

Same issue. `groq-api` and `groq-storyline` edge functions already exist server-side — frontend should call those.

**Fix:** Remove `VITE_GROQ_API_KEY` from Vite config entirely. Update all frontend callers to use `supabase.functions.invoke('groq-api', ...)`.

### 1.3 ✅ VITE_SENTRY_AUTH_TOKEN — FIXED (commit 5074c28)

### 1.4 ✅ No hardcoded secrets found
No literal `sk_live_`, `whsec_`, or JWT strings in source files. ✅

### 1.5 ✅ Runware API key correctly proxied
Uses `VITE_RUNWARE_PROXY_URL` → edge function. Correct pattern. ✅

### 1.6 🔴 TestCredits page publicly routed in production
**File:** `src/app/App.tsx`
```tsx
const TestCredits = lazy(() => import("@/app/pages/TestCredits"));
// Route: /test-credits — accessible to ALL users
```
This allows any user to trigger free credit grants. Must be guarded behind admin feature flag or removed.

**Fix:** Add `{user?.isAdmin && <Route path="/test-credits" element={<TestCredits />} />}` guard.

## 2. Dead Code / Stubs

### 2.1 🔴 billing.ts stub — unverified real checkout connection
**File:** `src/modules/subscriptions/billing.ts`

Previous audits flagged this as potentially returning fake checkout URLs. Status unconfirmed since last refresh. Must verify `createCheckoutSession()` calls `supabase.functions.invoke('create-checkout', ...)`.

**File:** `src/modules/subscriptions/` and post-generation handlers

The `brevo-email` Supabase edge function exists and is presumably functional. Frontend never calls it after checkout success or ebook generation. Users receive no confirmation emails.

**Files:** `e2e-test.mjs`, `e2e-test-v2.mjs`, `e2e-test-v3.mjs`

Three generations of E2E test scripts coexist. Only one is canonical. Others should be removed to avoid confusion.

## 3. DRY Violations

### 3.1 🟠 Auth token extraction repeated across components
Pattern `const session = await supabase.auth.getSession()` + null checks appears in 8+ components.

**Fix:** Centralize in `useAuth()` hook (may already exist — verify it's used everywhere).

### 3.2 🟠 Error toast pattern duplicated 15+ times
```ts
toast({ variant: "destructive", title: "Error", description: err.message })
```
Appears across nearly every component. Create `useErrorToast()` helper.

### 3.3 🟠 Supabase story/ebook fetch patterns duplicated
Gallery, Dashboard, and Admin all query the ebooks table independently with similar patterns.

**Fix:** Create `useUserEbooks(userId, options)` shared hook.

### 3.4 🟡 Feature flag checks inline
`if (import.meta.env.VITE_FEATURE_X === 'true')` scattered throughout. Should use `featureFlags.ts` config (which exists — verify usage).

## 5. Performance

### 5.1 🟡 Gallery + AdminUsers not lazy-loaded
**File:** `src/app/App.tsx`

These pages add to the initial bundle. Should use `React.lazy()` with `Suspense`.

### 5.2 🟡 AI-generated images not CDN-cached
Images generated by Runware are likely fetched fresh each time. Add Cloudflare or Supabase Storage CDN layer.

### 5.3 🟡 No service worker
No PWA/offline support. Mobile users on slow connections see blank pages during load. Add basic shell caching.
